Detailed Description

A stack protocol with methods for LIFO ("Last In, First Out") operations.

A stack is commonly compared to a stack of plates. Objects may be added in any order (-pushObject:) and the most recently added object may be removed (-popObject) or returned without removing it (-topObject).

- (BOOL) isEqualToStack: (id< CHStack >)  otherStack

Compares the receiving stack to another stack.

Two stacks have equal contents if they each hold the same number of objects and objects at a given position in each stack satisfy the -isEqual: test.

Parameters

- (NSEnumerator*) objectEnumerator

Returns an enumerator that accesses each object in the stack from top to bottom.

Returns
An enumerator that accesses each object in the stack from top to bottom. The enumerator returned is never nil; if the stack is empty, the enumerator will always return nil for -nextObject and an empty array for -allObjects.
Attention
The enumerator retains the collection. Once all objects in the enumerator have been consumed, the collection is released.
Warning
Modifying a collection while it is being enumerated is unsafe, and may cause a mutation exception to be raised.
